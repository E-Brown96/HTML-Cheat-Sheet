How to write forms in html

The tables below show how to set up and write forms into html. Forms are arguably the most difficult part of html but one of the most useful. This will also include login verfication.

For a form to be processed the computer needs a HTTP request to communicate/ interface with people. We need to give the form the location of where the forms information goes and what http request to make.

All forms have an action that determines where the information is sent.

Then the method is assigned a http verb.

The form tags

Tag Sub-Tag Name Description
<form> <form action="/example.html" method="POST"> Form tag This shows a basic form request with an action showing the file related to the form and a method which gives a html verb
<form(With sub elements)> <h1> or <p> Form sub tags These are sub tags that can be added with a form tag and are made up in the tags within the body shown earlier

To add text input into a form we need an input. The input will have a type element that determines how it reads.

The first input type is shown below in the table and the result is shown as an example below the table.

Tag Sub-Tag Name Description
<input> <input type="text" name="first-text-field"> Input tag When a form is submitted the entered text is sent to the action page and the value of name is the text
<input> <input type="text" name="text" value="text"> Input tag with name and value This shows the input with a name and value
<label> <label for="text"> Form label tag This adds a label to the form but note to add this to the input an id needs to be added to the input <input type="text" name="text id="text>

The form above was written using the tags shown in the table above but subsitutiing the word text for example and that was it. The last thing to do would be to hook the form up to a database at the minute it does not go anywhere.

Another input type is a password or a number etc. The table below shows a number of different input types.

Tag Sub-Tag Name Description
<input(password)> <input type="password" id="user-password name="user-password"> Input tag for passwords This shows the input for a password input type
<input(number)> <input type="number" name="text" id="text" step="1"> Input tag with for numbers This shows the input for a number input type, the step property is telling us that only intergers can be input
<input(range)> <input id="volume" name="volume" type="range" min="0" max="100" step="1"> INput tag with a range This will add an input range type in the form of a slider where we have set the min and max and the fact they must be integers

The table above is the general inputs and options for forms. Below is the table f more advanced options, this includes checkboxes, radio buttons, dropdown lists, datalists and text areas.

Tag Sub-Tag Name Description
<input(Checkbox)> <input id="cheese" name="topping" type="checkbox" value="cheese"> Input tag for checkbox This shows the input for a checkbox input type
<input(Radio buttons)> <input id="two" name="answer" type="radio" value="2"> Input tag with for radio buttons This shows the input for a radio button type
<input(dropdown list)> <select id="lunch" name="lunch"> <option value="pizza"> Pizza Input tag with a dropdown list This will add a dropdown list with the dropdown option being pizza
<input(datalist)> <input type="text" list="cities" id="city" name="city"> <datalist id="cities"> <option value-"text1"> Input tag for datalists This shows the input for a datalists, this will add a datalist with one option so far of text 1 which in this instance willbe a city
<input(textarea)> <textarea id="blog" name="blog" rows="5" cols="30"> Input tag for textarea This will add a textarea input with 5 rows and 30 columns
<input(submit button)> <input type="submit" value="Send"> Input tag for submit buttons This shows the input for a submit button
